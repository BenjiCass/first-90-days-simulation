AP build notes · BRN-02 Panel A
  • Four cost category cards stacked vertically. Each has a Yes/No toggle. Use object states in AP for the toggle interaction.
  • Learner must interact with all four toggles before the Continue button unlocks. Track with four boolean variables: cost_dev_flagged, cost_inf_flagged, cost_risk_flagged, cost_maint_flagged.
  • Risk cost card shown here in its flagged state for reference. Red border and flag indicator are the active state. Default state has no border emphasis.
  • cost_risk_flagged is the critical variable. Its value gates which consequence version surfaces on CNT-05. Store separately from the other three.
  • Panel A and Panel B are two states of one AP slide. Use layers or object show/hide to manage the transition between them. The Continue button triggers the layer change, not a slide navigation.

AP build notes · BRN-02 Panel B
  • Left column: three recommendation options. Right column: three justification options. Both columns must have a selection before Confirm unlocks.
  • Cost summary strip at top carries flag states from Panel A. Populated dynamically from the four boolean variables set in Panel A.
  • On confirm: set variable investment_recommendation = Approve, Reject, or Pilot. Lock all selections. Navigate to CNT-05.
  • Consequence routing on CNT-05 reads two variables in combination: investment_recommendation AND cost_risk_flagged.
  • Option 03 Pilot shown in teal selected state for reference. All three options use identical styling in default state.
  • Transformation Momentum and Organisational Trust both update on confirm. See variable architecture document for exact values per path.
  • No back navigation from this point. Decision is final once confirmed.
